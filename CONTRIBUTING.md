# Contributing to UOM BIT GPA Calculator 🎓

Thank you for your interest in contributing! We welcome all contributions that help make this tool more useful for the UOM BIT community.

---

### 🚀 Getting Started

To set up the project locally, follow these steps:
````markdown

git clone [https://github.com/YOUR-USERNAME/UOM-BIT-GPA-.git](https://github.com/YOUR-USERNAME/UOM-BIT-GPA-.git)
cd UOM-BIT-GPA
````

### 3\. Install Dependencies

```bash
npm install
```

### 4\. Run the Development Server

```bash
npm run dev
```

Open [http://localhost:3000](https://www.google.com/search?q=http://localhost:3000) in your browser.

-----

### 🌿 Creating a Branch

Create a new branch before making changes:

```bash
git checkout -b feature/your-feature-name
```

**Examples:**

  * `feature/add-dark-mode`
  * `fix/gpa-calculation-error`
  * `docs/update-readme`

-----

### 💻 Making Contributions

You can contribute by:

  * Adding new features (e.g., new semester support)
  * Fixing calculation bugs
  * Improving UI/UX (Tailwind/React)
  * Optimizing performance
  * Improving documentation
  * Refactoring code

-----

### ✅ Commit Guidelines

Write clear and meaningful commit messages:

```bash
git commit -m "feat: add semester GPA export feature"
```

**Recommended prefixes:**

  * **feat:** New feature
  * **fix:** Bug fix
  * **docs:** Documentation update
  * **style:** UI or styling changes
  * **refactor:** Code improvement
  * **chore:** Maintenance tasks

-----

### ⬆️ Push Your Changes

```bash
git push origin feature/your-feature-name
```

### 🔁 Creating a Pull Request

1.  Go to your forked repository on GitHub.
2.  Click **Compare & Pull Request**.
3.  Provide a clear description of your changes.
4.  Submit the Pull Request.

*Note: Your contribution will be reviewed before merging.*

-----

### 📌 Contribution Rules

  * Follow the existing project structure.
  * Use **TypeScript** best practices.
  * Keep components reusable and clean.
  * Ensure **responsive design** compatibility.
  * Test your changes thoroughly before submitting.

### 🚫 Important Notes

  * **Do NOT** commit `.env` or `.env.local` files.
  * **Do NOT** add unnecessary heavy dependencies.
  * Keep pull requests focused on a single change.

-----

### 💬 Need Help?

If you are unsure about a feature or improvement, please open an **Issue** first to discuss your idea.

Thank you for contributing ❤️
